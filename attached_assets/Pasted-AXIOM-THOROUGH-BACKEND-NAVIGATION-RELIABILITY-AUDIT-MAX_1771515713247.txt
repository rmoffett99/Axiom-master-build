PART B — UI NAVIGATION “DOUBLE CLICK” BUG (FIX + PROVE)
============================================================

Goal:
When a user clicks a nav item, route changes immediately on the FIRST click, every time.

B1) Identify root cause (choose the applicable pattern)
Common causes to check:
1) <a href="#"> or preventDefault interfering with router
2) Click handler on parent + child causing first click to toggle state, second to navigate
3) Link wrapped in button/div with onClick that steals focus
4) Mobile overlay / sidebar closing on first click and blocking navigation
5) Stale state due to setState + navigation order
6) Using window.location manually and blocking SPA routing

B2) Fix pattern rules (apply the correct one)
- If using Next.js: use <Link href="/path"> and do NOT wrap in an element that blocks clicks.
- If using React Router: use <NavLink to="/path"> or navigate() directly.
- Ensure ONLY ONE click handler owns navigation.
- If closing a drawer/menu, close it AFTER navigation OR in the same handler without preventing routing.

B3) Make navigation deterministic
Implement:
- One unified handler for nav clicks:
  - call router push first (or Link default)
  - then close menus/overlays
- Remove duplicate onClick layers and preventDefault unless required.

B4) Add an automated test (required)
- Add a Playwright/Cypress test:
  1) load app
  2) click nav item ONCE
  3) assert URL changes
  4) assert expected page content present
Repeat for all main nav routes.

B5) Manual reproduction checklist (required)
Provide:
- Before/after reproduction steps
- Confirmation the bug is gone on mobile + desktop viewports

Deliverable for Part B:
- Root cause explanation (1–3 sentences)
- Code diff summary
- Automated test proving single-click navigation works
